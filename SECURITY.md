# Security Policy

## Reporting a Vulnerability

THOX.ai takes security seriously. If you discover a vulnerability in this project or any THOX.ai project, please report it following these guidelines:

### Reporting Channels

- **Email**: security@thox.ai
- **GitHub Issues**: Open an issue on [thox-ai/thox-ai](https://github.com/thox-ai/thox-ai/issues)

### What to Include

When reporting a vulnerability, please include:

1. **Description**: A clear description of the vulnerability
2. **Steps to Reproduce**: How to trigger the issue
3. **Impact**: What the vulnerability allows an attacker to do
4. **Proof of Concept**: Code or screenshots demonstrating the issue
5. **Environment Details**: THOX.ai version, browser/OS, hardware specs
6. **Contact Information**: Your preferred way to be contacted

### What to Expect

- Your report will be acknowledged within **48 hours**
- We'll work to resolve the issue promptly
- You'll receive a notification when a fix is deployed
- For significant contributions, we may include you in release notes

### Security Best Practices

This project follows these security practices:

- **Input Validation**: All user inputs are validated and sanitized
- **API Key Handling**: API keys are properly stored and not hardcoded
- **HTTPS**: All API communications use secure HTTPS connections
- **Image Processing**: Images are processed securely to prevent vulnerabilities
- **Error Handling**: Errors are handled gracefully without exposing sensitive information

### Security Advisories

Security advisories will be published following the guidelines outlined in the [THOX.ai Security Guide](https://github.com/thox-ai/thox-meta/blob/main/docs/SECURITY.md).

## Security Contact

For security questions or concerns, please contact the THOX.ai security team at **security@thox.ai**.
